# Custom SSR

Мини-фреймворк для серверного рендеринга React-приложений, написанный с нуля на Express + Vite.
Реализует три стратегии рендеринга в стиле Next.js: **SSG**, **SSR** и **ISR** — с файловой маршрутизацией
и гидратацией на клиенте.

## Возможности

- **Файловый роутинг** — каждая страница живёт в `src/pages/<name>/index.tsx` и регистрируется через манифест.
- **Три стратегии рендеринга** (определяются автоматически):
  - **SSG** — страница без `getServerSideProps`. Рендерится один раз и кэшируется.
  - **SSR** — есть `getServerSideProps`, но нет `revalidateMs`. Рендерится на каждый запрос.
  - **ISR** — есть `getServerSideProps` и `revalidateMs`. Отдаётся из кэша, обновляется в фоне после устаревания.
- **Гидратация** — серверная разметка «оживает» на клиенте, состояние React работает (`useState` и т.д.).
- **Vite-сборка клиента** с хешированными ассетами и манифестом.
- **Безопасная передача данных** сервер → клиент через `<script type="application/json">` с экранированием.

## Технологии

- **React 19** (`renderToString` + `hydrateRoot`)
- **Express 5** — HTTP-сервер и маршрутизация
- **Vite 7** — сборка клиентского бандла
- **Tailwind CSS 4** — стили
- **TypeScript 5** (strict) + **tsx** для запуска сервера
- **axios** — HTTP-клиент для запросов данных

## Требования

- Node.js 20+
- pnpm

## Запуск

### 1. Установка зависимостей

```bash
pnpm install
```

### 2. Переменные окружения

Создайте файл `.env` в корне проекта:

```env
PORT=3000
# Интервал ревалидации ISR-страниц в миллисекундах (0 = выключить ISR)
ISR_REVALIDATE=10000
```

### 3. Режим разработки

Запускается в двух терминалах:

```bash
# Терминал 1 — сборка клиента с пересборкой при изменениях
pnpm dev:client

# Терминал 2 — сервер с автоперезапуском
pnpm dev:server
```

Приложение будет доступно на `http://localhost:3000`.

### 4. Продакшн-сборка

```bash
# Собрать клиентский бандл в public/
pnpm build

# Запустить сервер
tsx --env-file=.env src/server/index.ts
```

## Маршруты

| Путь          | Страница         | Стратегия |
| ------------- | ---------------- | --------- |
| `/`           | `pages/main`     | ISR / SSR |
| `/posts/:id`  | `pages/posts`    | SSR       |
| `/static`     | `pages/static`   | SSG       |

## Структура проекта

```
src/
├── app/
│   └── entry.client.tsx        # Клиентская точка входа: читает данные из DOM
│                               # и гидратирует нужную страницу
├── pages/                      # Страницы приложения (файловый роутинг)
│   ├── _pageManifest.ts        # Реестр страниц: путь → компонент + getServerSideProps + revalidateMs
│   ├── main/index.tsx          # ISR/SSR-страница
│   ├── posts/index.tsx         # SSR-страница с параметром :id
│   └── static/index.tsx        # Статическая (SSG) страница
│
├── server/
│   ├── index.ts                # Точка входа: поднимает HTTP-сервер
│   ├── app.ts                  # Сборка Express-приложения (static, /api, страницы)
│   ├── env.ts                  # Серверные переменные окружения (PORT, ISR_REVALIDATE)
│   ├── cache.ts                # In-memory кэш отрендеренного HTML
│   ├── render/
│   │   ├── render-page.tsx     # renderToString + вставка данных и ассетов в HTML
│   │   └── assets.ts           # Чтение Vite-манифеста, извлечение js/css страницы
│   └── routes/
│       ├── discover-pages.ts   # Регистрация роутов из pageManifest
│       ├── create-page-handler.tsx  # Ядро: выбор стратегии (SSG/SSR/ISR), кэш, рендер
│       └── api/index.ts        # Заготовка под API-роуты
│
└── shared/                     # Код, общий для сервера и клиента
    ├── api/                    # HTTP-клиент (axios) и запросы
    ├── config/                 # Константы (ID скриптов, PUBLIC_PATH) и env
    ├── html-template/          # <Html> — обёртка документа (head/body/ссылки на ассеты)
    ├── styles/                 # Точка входа Tailwind
    └── types/                  # Общие типы (Post, GetServerSideProps)
```

## Как это работает

1. **Сборка.** Vite собирает `entry.client.tsx` в `public/` и пишет `manifest.json` с именами хешированных файлов.
2. **Старт сервера.** `app.ts` раздаёт статику из `public/`, подключает `/api` и регистрирует страницы из `_pageManifest.ts`.
3. **Запрос страницы.** `create-page-handler` определяет стратегию, при необходимости вызывает `getServerSideProps`,
   рендерит React в строку и оборачивает в `<Html>` с подключёнными js/css.
4. **Передача данных.** Начальные пропсы и имя страницы встраиваются в HTML как JSON-скрипты.
5. **Гидратация.** На клиенте `entry.client.tsx` читает эти данные, динамически импортирует нужную страницу
   и вызывает `hydrateRoot`.

## Добавление новой страницы

1. Создайте `src/pages/<name>/index.tsx` с дефолтным экспортом компонента.
2. При необходимости экспортируйте `getServerSideProps`.
3. Зарегистрируйте страницу в `src/pages/_pageManifest.ts`.
